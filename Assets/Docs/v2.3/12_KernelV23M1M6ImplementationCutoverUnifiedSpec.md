# 12_KernelV23M1M6実装差し替え統合仕様書

最終更新日: 2026-05-23
適用範囲: M1 〜 M6
文書種別: 実装実行仕様（ゲート証跡専用文書ではない）

## 1. 目的

本仕様の目的は、現行の巨大な実行基盤を段階的に完全差し替えすることである。
差し替え完了後の許可実行経路は、次を必須とする。

- 検証済み計画に基づく起動・登録・構築・活性化
- 明示宣言された依存関係のみを使用
- 失敗時は fail-closed
- 旧互換面は直列化継続専用かつ非権限

## 2. 非目的

- 証跡文書の作成のみで完了扱いにすること
- 旧経路を残したまま「動くから許可」と判定すること
- 動的探索・暗黙補完・静かなフォールバックを許可すること

## 3. 用語

- 許可実行経路: 本仕様で許可された唯一の実行権限経路
- 旧経路: 旧設計に由来する権限経路、探索経路、互換経路
- 互換面: 直列化継続などの移行都合で一時的に残す面
- 完全差し替え: 旧経路が許可実行経路から物理的に除去された状態

## 4. M1〜M6 実装マイルストーン

### M1 実態棚卸しと凍結

実装作業:
- 旧権限経路・旧探索経路・旧フォールバック経路をコード単位で列挙
- 置換先（新系契約）を各経路へ1対1で割当
- 凍結対象を明示し、実装差し替え中の拡張を禁止

完了条件:
- 旧経路一覧に欠落がない
- 置換先未定義の経路がない

### M2 中核 command 面の強制切替

実装作業:
- command 実行入口を検証済み実行セッション必須へ統一
- 旧 catalog/旧 discovery/旧 fallback 入口を削除
- 拒否コードと診断情報を明示し、無音継続を禁止

完了条件:
- 検証済み実行セッションなしで command 実行不可
- 失敗時は明示診断を必ず返す

### M3 葉スコープ降格と互換境界固定

実装作業:
- 葉スコープから旧権限経路を除去
- 互換シェルを直列化継続専用へ制限
- 参照継続を保ちつつ実行権限を新系へ移動

完了条件:
- 互換面が権限源にならない
- 参照切れなしで旧権限経路が不達

### M4 ルートシーン統合切替

実装作業:
- ルート起動を計画先行契約へ一本化
- 登録・構築・活性化の順序を決定論化
- 旧ルート回り込み経路を削除

完了条件:
- ルートの順序ずれが起きない
- 旧回り込み経路が到達不能

### M5 旧系削除・ハードニング・性能確定

実装作業:
- 旧権限経路を物理削除
- 拒否分類と診断情報を実行で確定
- 性能予算を基準値比較で判定
- 互換面の退役/残置を明示規則で閉じる

完了条件:
- 旧権限経路が許可実行経路に存在しない
- フォールバック観測なし
- 性能違反未解決なし

### M6 完全証明と公開判定

実装作業:
- M1〜M5 の実行証明を統合
- 網羅率・残留リスク・独立レビュー結果を確定
- 公開可否を明示決定

完了条件:
- 完全差し替え証明が承認済み
- 未解決ブロッカーが0

## 5. 旧経路削除の強制規則

- 旧経路は「無効化」ではなく「物理削除」を原則とする
- 一時残置は期限・退出条件・担当を必須化
- release 相当プロファイルで旧実行アダプタは許可しない

## 6. 実装順序（固定）

1. 旧探索/旧フォールバックの遮断
2. ルートと command 入口の強制切替
3. 葉・互換面の非権限化
4. 旧経路の物理削除
5. 性能・診断・継続性の閉鎖確認
6. 完全証明と公開判定

## 7. 必須検証

各マイルストーンで次を必須とする。

- 変更ファイル一覧（削除/置換の根拠付き）
- 変更対象のコンパイル検証
- fail-closed を示す対象試験
- 旧経路不達の実行観測
- 参照継続の確認（必要対象のみ）

## 8. ブロック条件

次のいずれかが残る場合、次マイルストーン開始を禁止する。

- 旧権限経路の到達性が残存
- 必須拒否コードまたは診断項目の欠落
- フォールバック観測あり
- 性能違反が未解決
- 互換面の期限・退出条件未設定

## 9. 完了定義

M1〜M6 完了は次を同時に満たした場合のみ成立する。

- 旧実行アーキテクチャが許可実行経路から物理的に除去済み
- 新実行アーキテクチャが計画駆動・明示依存・fail-closedで稼働
- 必須互換面が直列化継続専用かつ非権限
- 実装証明・性能証明・独立レビューが承認済み
